📋 Dify Mapping Notes

  • Each agent = one Dify Workflow. Create three separate workflows: intake, planner, tracker.
  • LLM nodes (purple) → Dify LLM block. Paste the corresponding system prompt and wire in context variables.
  • I/O nodes (cyan) → Dify HTTP Request or Code block calling GitHub REST API / S3 API.
  • Conditional router (amber) → Dify IF/ELSE or Question Classifier block. Intake: check if READY_TO_SCAFFOLD signal is present in LLM output or clarification_turns ≥ 3. Tracker: check if error field is non-null.
  • Human-in-the-loop pause (dashed) → In Dify, use a Conversation workflow (not a one-shot Workflow). The loop is the Dify conversation itself; each Slack reply re-enters the flow at the LLM (clarify) node. Use a variable to count turns.
  • JSON parsing (scaffold_skill_tree / generate_tasks) → Dify Code block (Python) to strip ```json fences and parse.
  • State persistence (clarification_turns, skip_state) → Dify Variables (conversation-level for Intake, workflow-level for Planner).
  • Planner trigger → Set the Planner workflow as a Dify Scheduled Task (daily cron).
