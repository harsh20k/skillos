Deploying my first full-stack app on AWS felt like assembling furniture without the manual. Terraform created the pieces; I spent days figuring out the order.

IAM quota hit—too many managed policies on one user. Fix: a single inline policy for CodeCommit, CodeBuild, CodePipeline. Then the backend kept exiting. Tables didn't exist yet; RDS was empty and the app expected them. One-off create_tables, then start the container. Another gotcha: the DB password had special characters. Without urlencode in the connection string, asyncpg said "password authentication failed" and I lost an hour.

Theory is clean. The first deploy is messy.

What blocked you the longest on your first cloud deploy?

# outline
- Hook: Terraform created the pieces; ordering and gotchas were the real work
- IAM: managed policy quota → inline policy for CI/CD
- Backend/RDS: empty DB, app expects tables → create_tables before container start
- DATABASE_URL: special chars in password → urlencode or auth fails
