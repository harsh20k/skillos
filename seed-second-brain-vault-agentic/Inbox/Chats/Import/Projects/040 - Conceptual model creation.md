## 📋 Requirement Statement (Simplified)

- Build a **distributed database system (DDBMS)** to store and manage fish presence data.  
- Combine data from **different sources** — DFO surveys, OTN telemetry, and provincial records.  
- Include main entities: **Taxon**, **Event**, **Location**, **Occurrence**, plus optional ones like Catch or Telemetry.  
- Allow users to **search and filter** data by fish species, location, and time.  
- Keep data **consistent and synchronized** across all distributed parts of the system.  
- Support **analytics and AI/ML models** under the DeepSense program.  
- Follow **Darwin Core standards** for easy data sharing with global databases like OBIS and GBIF.
