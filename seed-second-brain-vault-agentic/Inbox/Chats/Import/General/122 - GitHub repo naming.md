# GitHub repo naming

**Priority:** Medium

## Conversation

### Assistant

Yes, you can name a GitHub repository something like **`compliment-generator`** or **`ComplimentGenerator`**, but there are a few common naming conventions people usually follow:  

### ✅ Common GitHub Repo Naming Styles:
1. **kebab-case (most common, recommended)**  
   👉 `compliment-generator`  
   - Easy to read.  
   - Works well with URLs (GitHub repos are often used in URLs).  

2. **camelCase (less common, but used sometimes)**  
   👉 `complimentGenerator`  
   - Looks more like variable naming in code.  

3. **PascalCase (used occasionally for libraries/apps)**  
   👉 `ComplimentGenerator`  
   - Sometimes used for project names or when you want it to look like a product.  

### 🚫 Avoid
- Spaces: `compliment generator` ❌ (GitHub replaces with dashes anyway)  
- Special characters: `compliment@gen` ❌ (can cause issues in URLs)  

👉 Best practice: Use **`compliment-generator`** (kebab-case).
