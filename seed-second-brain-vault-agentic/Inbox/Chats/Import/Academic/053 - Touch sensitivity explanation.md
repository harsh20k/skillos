### **Option A: Quick “One-Day” Project (Today)**
If you want to keep momentum and not lose a week, you can do something really small but polished in just a few hours. Examples:
- **Lecture Notes Parser**: A simple app that takes plain text (like copied notes) and auto-formats into sections, keywords, and summaries.  
- **Timetable Generator**: Input your class schedule → outputs a clean calendar view or even a `.ics` file.  
- **“Hello Halifax” Map Project**: Pull in open data (bus stops, cafes, libraries) and make a small interactive map.

These are quick to execute, so you have something to share before classes even begin.

---

### **Option B: “Class-Inspired” Project (From Tomorrow)**
If classes do start tomorrow, you can wait and build something tied directly to what you learn. That gives your projects a story: *“This week in X course, I learned Y, so I made Z.”*  
Examples:
- If you cover **algorithms/data structures** → build a visualizer for that algorithm.  
- If you start with **databases** → make a small app with SQLite and a clean UI.  
- If it’s **web stack basics** → build a minimal tool using what you’re taught that week.  

This approach makes your projects directly aligned with your MACS journey, which looks strong on LinkedIn and to employers.
