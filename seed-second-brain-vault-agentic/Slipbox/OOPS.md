# OOPS (Object-Oriented Programming System)

**Object-Oriented Programming System (OOPS)**, commonly known as **Object-Oriented Programming (OOP)**, is a [[Programming Paradigm]] that uses “objects” to represent and model real-world entities. It emphasises structuring software around objects rather than actions or logic.

## Key Concepts

1. **[[Classes and Objects#Object?|Objects]]**:  
   These are instances of classes. An object contains both data (attributes) and behaviours (methods) that model a real-world concept or entity.

2. **[[Classes and Objects#Class|Classes]]**:  
   A class is a blueprint that defines the structure and behaviours of an object. It specifies which attributes and methods its instances (objects) will have.

3. **[[Encapsulation]]**:  
   [[Encapsulation]] means bundling data (attributes) and code (methods) together, and restricting direct access to some of the object’s components. This protects the object’s internal state and makes it easier to modify and maintain code.

4. **[[Inheritance]]**:  
   [[Inheritance]] allows one class (subclass or child class) to derive properties and methods from another class (superclass or parent class). This promotes code reuse and hierarchical class structures.

5. **[[Polymorphism]]**:  
   [[Polymorphism]] enables objects to be treated as instances of their parent class rather than their actual class. Different classes can implement the same methods in different ways, allowing one interface to handle multiple underlying forms.

6. **[[Abstraction]]**:  
   [[Abstraction]] involves showing only the necessary features of an object and hiding the implementation details. This reduces complexity and enhances the clarity of the code.

## Why Use OOP?

- **Modularity and Reusability**:  
  By modelling real-world entities as objects, it’s easier to break down complex problems and reuse code across projects.
  
- **Maintenance and Extensibility**:  
  [[Encapsulation]] and [[Abstraction]] improve code maintenance, while [[Inheritance]] and [[Polymorphism]] make it simpler to extend existing code.

- **Improved Reliability**:  
  With well-defined [[Classes and Objects]], code is generally more robust, easier to test, and less prone to errors.

## Examples of OOP Languages

- **[[Java]]**: Designed from the ground up for OOP.
- **C++**: An extension of C that supports OOP.
- **[[Python]]**: A multi-paradigm language that supports OOP.
- **C#**: A language developed by Microsoft, strongly based on OOP principles.
- **[[Swift]]**: A multi-paradigm language that also supports OOP.

## Conclusion

OOPS (Object-Oriented Programming System) is about thinking of a program as a collection of interacting objects. By using classes, [[Encapsulation]], [[Inheritance]], [[Polymorphism]], and [[Abstraction]], OOP fosters cleaner, more organised, and more scalable code.
